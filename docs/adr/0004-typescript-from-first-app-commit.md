# TypeScript from the first app commit

Application code for the MVP will be written in TypeScript from the first app commit rather than starting in plain JavaScript and migrating later. The TypeScript configuration should use strict mode plus extra safety flags such as `noUncheckedIndexedAccess` and `exactOptionalPropertyTypes`, with unsafe third-party boundary casts isolated behind adapters. The product's core contracts are data-model-heavy and persistence-sensitive: Notebook, Section, Page, Page Type, Canvas Item, Canvas Region, persistence adapters, generated metadata, and future AI enrichment all need explicit types before they become stored browser data. Plain JavaScript is acceptable for generated artifacts or tool output, but not as the authored application source.
