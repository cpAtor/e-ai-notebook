# Excalidraw-native canvas foundation

The Drawing Screen will use Excalidraw as its canvas foundation rather than tldraw, because tldraw v4/v5 require a production license key and the prior tldraw-shaped foundation is no longer viable for a private deployed Notebook. Excalidraw is a foundational editor dependency, not a thin renderer: Notebook behavior must integrate through a deep Notebook Canvas module that preserves native Excalidraw desktop and mobile affordances, especially text editing, selection, styling controls, default colors, and canvas search. App composition should pass Page, Canvas Items, Canvas Regions, theme, Notebook actions, and change callbacks into Notebook Canvas; Excalidraw element formats, sync scheduling, native UI wiring, and persistence mapping should remain inside that module.
